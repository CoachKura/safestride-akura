🔧 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════

ISSUE 1: Git push fails with authentication error
───────────────────────────────────────────────────
SOLUTION:
- Create GitHub Personal Access Token
- Go to: https://github.com/settings/tokens
- Generate new token (classic)
- Scopes: repo (full control)
- Copy token
- Use as password when pushing

ISSUE 2: Cloudflare build fails
───────────────────────────────────────────────────
SOLUTION:
- Check Build settings
- Ensure "Build output directory" is /public
- Ensure public/ folder exists in repo
- Re-deploy manually

ISSUE 3: Pages show 404 errors
───────────────────────────────────────────────────
SOLUTION:
- Check file names match URLs
- Ensure .html extension in URLs
- Example: /onboarding.html NOT /onboarding
- OR configure _redirects file for clean URLs

ISSUE 4: Chart.js doesn't load in production
───────────────────────────────────────────────────
SOLUTION:
- Check browser console (F12)
- Verify CDN URL: https://cdn.jsdelivr.net/npm/chart.js@4.4.0
- Check for CORS or CSP issues
- Ensure internet connection on device

ISSUE 5: Supabase connection fails in production
───────────────────────────────────────────────────
SOLUTION:
- Check Supabase URL is correct
- Verify anon key is valid
- Ensure CORS is enabled in Supabase dashboard
- Check if RLS policies allow anonymous access
